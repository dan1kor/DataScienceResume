# Избранные проекты

## Match Outcome Prediction: Dota 2

Разработка модели для предсказания победителя в матчах на основе игровых логов. Проект охватывает путь от сырых данных до оптимизированного ансамбля моделей.

### Обзор
Основная задача — классификация исхода матча (Radiant win / Dire win). Проект разделен на исследовательскую часть (baseline) и этап улучшения модели с применением продвинутого Feature Engineering.

### Структура проекта

* `notebooks/`
    * `base.ipynb`: Первичный анализ данных (EDA), предобработка и построение базовой логистической регрессии.
    * `advanced.ipynb`: Использование градиентного бустинга, оптимизация гиперпараметров и оценка финальных метрик (ROC-AUC, Accuracy).
* `src/`
    * `transformers.py`: Кастомные классы-трансформеры (совместимые с Scikit-learn Pipeline) для автоматизации обработки признаков и инженерии фичей.

### Данные & Модели
* **Dataset:** В проекте используется датасет с **Kaggle**. Из-за большого объема данных файлы не включены в репозиторий. Вы можете скачать их [здесь](https://www.kaggle.com/competitions/dota-2-hse-ml-1-course-competition-2026/data) и поместить в директорию `data/raw`.
* **Models:** Обученные веса моделей и промежуточные артефакты доступны в [облачном хранилище](https://drive.google.com/drive/folders/1kZW71o90sdF80me7wuOKLGC3EvxOJUWI?usp=sharing). Скачайте необходимые `.pkl` файлы и поместите их в папку `models/` для локального запуска предсказаний.

### Стэк
* **Language:** Python
* **Libraries:** Pandas, NumPy, Scikit-learn, Matplotlib, Seaborn.
* **Concepts:** Binary Classification, Feature Engineering, Cross-Validation, Hyperparameter Tuning (Optuna).

### Основные фичи
* Реализована кастомная обработка временных рядов (события внутри матча).
* Созданы производные признаки (Gold/XP advantage, Hero synergies).
* Весь препроцессинг вынесен в отдельные скрипты для обеспечения воспроизводимости и чистоты кода.

## Исследование миграционных траекторий в Арктической зоне РФ (АЗРФ)

Аналитический проект по изучению факторов оттока населения в малых арктических поселениях. Демонстрирует навыки работы с "грязными" реальными данными и проверку сложных статистических гипотез.

### Описание проекта

Цель работы — выявить, почему при схожих климатических условиях арктические муниципалитеты демонстрируют разные темпы оттока населения. 

* **Объект исследования:** Муниципальные образования АЗРФ.
* **Методология:** Индексный метод, нормализация социально-экономических показателей, проверка статистических гипотез о гетерогенности эффектов удаленности.

### Структура проекта
* `main.ipynb`: Основной рабочий ноутбук. Включает в себя:
    * Загрузку и первичную очистку данных.
    * Расчет агрегированных индексов (рыночный доступ, инфраструктура).
    * Визуализацию распределений и корреляционный анализ.
    * Проверку четырех ключевых гипотез исследования.
* `data/`: (Директория синхронизирована с Google Drive)
    * `raw/`: Исходные данные из открытых источников (Росстат, СберИндекс, муниципальная статистика 2023 г.).
    * `processed/`: Очищенные данные, приведенные к единому формату, и рассчитанные индексы.

### Данные
Ввиду специфики и объема используемых данных, директория `data/` не включена в репозиторий. Все наборы данных (сырые и обработанные) доступны по [ссылке](https://drive.google.com/drive/folders/1jpeW0odZrCc9ZvPrtRj69lML_xTTw-rl?usp=sharing)

Для воспроизведения анализа скачайте содержимое и сохраните в соответствующую папку проекта.


### Стек технологий
* **Python:** Основной язык анализа.
* **Библиотеки:** Pandas (обработка данных), NumPy (вычисления), Matplotlib / Seaborn (визуализация), Scipy / Statsmodels (статистический анализ).

### Ключевые выводы

* Подтверждена значимая роль рыночного доступа для удаленных территорий.

* Выявлены «кризисные кластеры» поселений, требующие приоритетного государственного внимания.

* Арктический фактор сам по себе создает меньшую вариативность, чем фактор логистической удаленности от рынков.


## Smart Nutritionist: Recipe Analysis & Rating Forecast
Интеллектуальное консольное приложение для анализа состава блюд и прогнозирования их рейтинга на основе машинного обучения. Проект охватывает полный цикл: от сбора данных через внешние API до реализации логики в виде Python-модуля.

### Ключевые особенности
* **ML-прогнозирование:** Предсказание категории качества блюда (Bad, So-so, Great) на основе комбинации ингредиентов.
* **Анализ нутриентов:** Интеграция с внешними базами данных для расчета КБЖУ и доли от дневной нормы потребления.
* **Рекомендательная система:** Поиск топ-3 наиболее похожих рецептов с использованием алгоритма ближайших соседей (K-Nearest Neighbors).
* **Архитектура:** Чистый код с использованием объектно-ориентированного программирования (ООП).

### Структура проекта
* `nutritionist.py` — Главный исполнительный скрипт (CLI).
* `recipes.py` — Модуль с классом `Recipe`, инкапсулирующий логику предсказания и поиска.
* `notebooks/`
    * `recipes.ipynb` — Исследование моделей: регрессия (RMSE), классификация и кастомные метрики оценки. Обучение KNN для рекомендаций.
    * `nutrition_facts.ipynb` — Скрипт взаимодействия с API для сбора и трансформации данных о пищевой ценности.
* `data/` — Обработанные датасеты и CSV с нутриентами.
* `models/` — Сохраненные дампы обученных моделей (`.pkl`).

## Стек технологий
* **Core:** Python (Classes, Methods, Sys).
* **DS Stack:** Pandas, NumPy, Scikit-learn (SVC, Logistic Regression, KNN).
* **Data Engineering:** REST API (Requests), JSON processing, Joblib для сериализации моделей.
