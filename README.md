# Избранные проекты

## 1. Match Outcome Prediction: Dota 2 (ML Pipeline)
Разработка модели для предсказания победителя в матчах на основе игровых логов. Проект охватывает путь от сырых данных до оптимизированного ансамбля моделей.

### Основные фичи:
* **Feature Engineering:** Реализована кастомная обработка временных рядов (события внутри матча) и создание производных признаков (Gold/XP advantage, Hero synergies).
* **Architecture:** Весь препроцессинг вынесен в отдельные скрипты (`src/transformers.py`) для обеспечения чистоты кода и воспроизводимости.
* **Optimization:** Оптимизация гиперпараметров с использованием **Optuna**.

### Ссылки:
* **Dataset:** [Kaggle Competition Data](https://www.kaggle.com/competitions/dota-2-hse-ml-1-course-competition-2026/data) (поместить в `data/raw`).
* **Models:** [Веса моделей на Google Drive](https://drive.google.com/drive/folders/1kZW71o90sdF80me7wuOKLGC3EvxOJUWI?usp=sharing).

## 2. Исследование миграции в Арктической зоне РФ (Data Analytics)

## 3. Smart Nutritionist (Production ML / CLI App)
Консольное приложение для анализа нутриентов и прогнозирования рейтинга блюд. Акцент на архитектуре кода и бизнес-ориентированном выборе метрик.

### Технологические решения:
* **Full Cycle:** От сбора данных через REST API до реализации логики в виде Python-модуля с использованием **ООП**.
* **Recommendation System:** Поиск похожих рецептов с использованием **K-Nearest Neighbors (KNN)**.
* **Risk-Driven ML:** Выбор метрик классификации (Precision/Recall) с учетом стоимости ложных прогнозов (Cost-sensitive learning).

## Инженерный подход к ML
В проекте реализован осознанный выбор метрик. Вместо стандартной Accuracy использован подход оценки стоимости ошибки (Cost-sensitive learning):
* **Приоритет:** Минимизация ситуаций, когда плохое блюдо классифицируется как «Шедевральное» (проблема ложных ожиданий пользователя).
* **Моделирование:** Сравнение производительности классических моделей и ансамблей (Voting Classifier).
